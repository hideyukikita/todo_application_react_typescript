# Todoアプリ 基本設計書
## 1.1 システム概要
- **フルスタック構成**: Docker上で稼働する、ReactとTypeScript、Node.jsを用いた高機能ToDo管理アプリ。
- **マルチデバイス対応**: WSL2環境からネットワークを開放し、同一Wi-Fi内のPCおよびスマートフォンからリアルタイムにタスク管理が可能。
- **型安全な開発**: Zodを用いた「共通スキーマ」により、フロント・バック・DB間でのデータ整合性を保証。

## 1.2 ターゲット・利用シーン
- 学習記録、家事、仕事のタスクを詳細（メモ・優先度・締切）に管理したい。
- 家族や自分自身が、異なるデバイスから同一のタスクリストにアクセスする。

## 1.3 機能要件
1. ユーザー認証
   - 新規登録・ログイン（JWTによる認可）。
   - パスワードのハッシュ化（bcrypt）。
   - `AuthContext` によるログイン状態の全域管理。
2. ToDo管理（論理削除対応）
   - タスクのCRUD（作成・取得・更新・論理削除）。
   - 優先度設定（3段階）、締切日設定、メモ機能。
   - 完了ステータスの1クリック切り替え。
3. 表示・分析
   - 優先度順、締切日順のソート機能。
   - Rechartsを用いた「完了・未完了比率」と「日別完了数」の可視化。
4. ネットワーク
   - 同一Wi-Fi内の別のデバイスからのアクセス許可設定。
